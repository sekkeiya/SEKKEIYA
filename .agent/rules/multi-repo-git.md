# 複数リポジトリ横断 Git 運用ルール

このワークスペースには以下の3つのリポジトリが存在します。
1. sekkeiya (司令塔 / メイン)
2. r3dm-share (共有ライブラリ)
3. 3d-shape-layout (レイアウトアプリ)

## ブランチ操作
- 横断改修（Refactoring等）を行う際は、原則として3つのリポジトリでブランチ名を統一すること。
- 修正が不要なリポジトリについてはブランチを作成せず、mainのままにするか、何もしないこと。

## push と PR 作成
- 各リポジトリのルートディレクトリに移動して git 操作を行うこと。
- PR を作成する際は、`sekkeiya/.agent/rules/pr-writing.md` の基準に従うこと。
- GitHub CLI (gh) が利用可能な場合は、PR の作成まで自動化を試みること。
