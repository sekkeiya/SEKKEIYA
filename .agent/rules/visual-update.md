# visual-update

大きなUI改修や見た目に関わる変更を行った場合は、コード修正だけで終わらず、必ず最新の見た目も記録してください。

## 対象
以下のような変更が対象です。

- レイアウト変更
- デザイン変更
- ボタン、カード、モーダル、ヘッダー、サイドバーなどのUI変更
- ダッシュボード、一覧画面、詳細画面などの見た目変更
- 新規ページ追加
- レスポンシブ対応
- アニメーションや表示状態の改善
- Before / After が視覚的に伝わる変更

## 必須手順
1. ローカル環境で対象画面をブラウザ表示して確認する。
2. 変更内容が分かる状態でスクリーンショットを撮影する。
3. スクリーンショットを `docs/latest_update.png` として保存する。
4. 画像ファイルを含めて commit / push する。

## ファイルルール
- 保存先は原則 `docs/latest_update.png`
- 既存ファイルがある場合は最新画像で上書きしてよい
- 1枚で伝わりにくい場合は追加で以下の形式も許可する
  - `docs/latest_update_01.png`
  - `docs/latest_update_02.png`

## 撮影ルール
- UI全体が分かるように撮影する
- 重要な変更箇所が見切れないようにする
- 不要なデバッグ表示や開発用UIは極力消す
- 個人情報や秘密情報が写らないようにする
- 可能であれば実際の利用状態に近い画面を使う

## push前チェック
以下を確認してから push すること。

- 画面が正しく表示されている
- 変更内容が画像で分かる
- `docs/latest_update.png` が追加または更新されている
- 画像ファイルを git に含めている

## commitメッセージ推奨
UI変更時は、必要に応じて以下のように画像更新も含むことが分かるメッセージにする。

- `UI改善と最新スクリーンショット更新`
- `ダッシュボード改修とdocs/latest_update.png更新`
- `レスポンシブ調整 + visual update`

## 補足
大きなUI改修では、スクリーンショット更新も作業の一部です。
コードだけでなく、最新の見た目が追跡できる状態を維持してください。
