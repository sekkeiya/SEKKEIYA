# Project / Board Architecture

## 目的
SEKKEIYA エコシステムにおけるデータ構造の基本骨格である「Project」「Board」「Item」「Asset」の役割と責務境界を定義します。
このアーキテクチャは、各子アプリが独立した独自のデータ構造を持たず、SEKKEIYAの共通規約に従って相互連携するための基盤となります。

## 基本概念 (現時点の仕様)
SEKKEIYAのデータ構造は、最上位から順に **Project > Board > Item ( > Asset )** の階層構造を取ります。
ただし、**現フェーズでは Board を最小の実行コンテキスト（実行単位）として先行実装**しており、Project は将来的な上位の管理単位として段階的に導入していく方針をとっています。

## Project の責務 (将来構想・段階導入)
* **役割**: 上位の所有者・権限管理・リソース整理の単位。
* **位置付け**: 複数の Board を束ねるフォルダのような役割を果たし、大規模な案件やチームごとの管理境界として機能します。
* **現状**: 現時点では実装を必須とせず、将来的な段階導入（フェーズ10以降の拡張）を見据えたプレースホルダー的な立ち位置です。

## Board の責務 (現在の仕様)
* **役割**: SEKKEIYA OS における**「最小の実行コンテキスト」**。
* **位置付け**: ユーザーの実際の作業やディスカッションが行われる「空間・キャンバス」です。
* **特徴**: 
  * チャット (AI Chat) や ドライブ (AI Drive)、および各子アプリは、常に「特定の Board を開いている（`boardId` を持つ）」状態で起動し、そのコンテキスト内で動作します。
  * 共同編集や閲覧の権限は、現時点ではこの Board 単位で制御されます。

## Item の責務 (現在の仕様)
* **役割**: Board 内に配置される、具体的な「要素・オブジェクト」の単位。
* **種類**: `model` (3Dモデル), `layout` (空間レイアウト), `image`, `text`, `pdf` など。
* **特徴**: 
  * 子アプリはそれぞれ得意とする Item Type を持ち（後述）、SEKKEIYA共通の Adapter を介して Item を操作します。
  * Item 自身が実体データを持つ場合と、外部の Asset を参照する場合があります。

## Asset の責務 (現在の仕様)
* **役割**: 物理的な大容量ファイルや、不変のバイナリデータの実体（Storage に保存される `.obj`, `.glb`, `.png` など）を管理するレコード。
* **特徴**: 
  * 複数の Item から参照される再利用可能なリソースです（例：1つの「椅子」Asset を、複数のレイアウト Board の Item が参照する）。

## Project と Board の導入順序
1. **先行実装**: 現在は **Board** を中心としたアーキテクチャとしてシステムが稼働しています。「Board = 実行コンテキスト」としての基盤（`BoardContext`）を先に完成させています。
2. **段階導入**: 今後、権限管理やファイルの整理が複雑化した段階で、上位概念である **Project** を本格導入します。設計上は既に `projectId` を受け入れる空間を確保してあります。

## 子アプリとの関係
* 各子アプリ（3DSS, 3DSL, 3DSP等）は、**独自にルートレベルのDB構造（独自Schema）を持ちません**。
* すべての子アプリは、この SEKKEIYA 標準の「Project / Board / Item」構造の上に「UI / 処理」として乗っかる形で動作します。

## BoardContext との関係
* `BoardContext` は、現在開いている Board の `boardId` (将来は `projectId` も) を保持する **SEKKEIYA OS のグローバル状態**です。
* URLパラメータ (`?boardId=xxx`) や、`global-panel` のストアを通じて、どの子アプリに画面遷移してもこの実行コンテキストが維持・復元されます。

## 今後の拡張方針
* Project を使った本格的なチーム・エンタープライズ権限管理の導入。
* Item の型 (`type`) 追加による新しい子アプリ（3DSQ等）のプラグイン的統合。
* 複数の Board をまたいだグローバル検索（Drive層拡張）の実装。
