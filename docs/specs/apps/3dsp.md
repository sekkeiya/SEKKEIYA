# 3DSP (3D Shape Presents)

## 役割
3DSP（3D Shape Presents）は、構築されたモデルやレイアウト、および付随するテキストや画像素材を用いて、他者へ意図や世界観を説明するための**「伝えるための構成・プレゼンアプリ」**です。

## 主な画面
* プレゼンテーション・ボード エディタ (ブロック単位での構成画面)
* 閲覧者(Viewer)用スライド・ショーケース画面
* カスタムテーマ・テンプレート設定画面

## 主に扱う Item / Asset
* Board 内の `model`、`layout` に加えて、**`text` (説明文), `image` (スクリーンショットやパース), `pdf` 等の多様なメディア Item** を扱います。
* 上記の成果物をまとめて提示（パッケージング）する役割を持ちます。

## 読むデータ
* `boards/{boardId}` コンテキスト情報。
* 3DSS や 3DSL で作成された成果物となる全パターンの Item。

## 書くデータ
* プレゼンテーションの「スライド（シーン）」ごとの順序・カメラ設定・注釈テキストなどを `presentation` 型の Item または Board 自体のメタ属性として保存します（現在設計中）。

## Project / Board との関係
* 通常の作業用 Board 内の成果物を最終整理するレイヤーとして機能します。
* 将来的には、1つの Project 内に存在する複数の Board（検討プロセス）から、最良の成果物を抽出して 3DSP 用の公開 Board に集約するといった上位関係の構築が見込まれます。

## AI Drive との関係
* 作成済みのすべての素材（モデル、レイアウトシーン、参考画像）を AI Drive から呼び出し、プレゼンボードの任意の場所に配置します。

## AI Chat との関係
* 「このレイアウトに基づくコンセプト文を作成して」「顧客向けのプレゼン構成案を考えて」といった、ドキュメンテーションやライティングの壁打ち相手として AI Chat を活用します。

## 3DSS / 3DSL との関係
* 3DSS（部品）と3DSL（構成）で作られた成果を**「再編集・再構成する（ビューを作る）」**位置付けです。
* そのため、3DSP 上でモデルそのものを新規に作ったり、詳細な空間配置をミリ単位で修整するUIは持ちません。見せ方に特化します。

## 将来の責務境界
* 3DSP は「誰に・どうやって魅せるか」に集中します。共有リンクの発行ログ、閲覧者のアクセス解析、注釈（コメント）機能などを強化し、外部顧客とのフロントエンド接点（納品物クライアント）として成長します。
