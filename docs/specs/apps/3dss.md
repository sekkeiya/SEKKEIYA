# 3DSS (3D Shape Share)

## 役割
3DSS（3D Shape Share）は、単体の3Dモデルに対するプレビュー、属性管理、整理、共有に特化した**「モデル特化の閲覧・管理・共有アプリ」**です。

## 主な画面
* トップページ / 新規アップロード・インポートUI
* ダッシュボード画面（モデル一覧検索機能、ボード切り替え）
* モデル詳細画面・ビューア（Webでのリアルタイム閲覧、テクスチャや材質の調整）
* 左サイドバー（カテゴリー）および右サイドバー（モデル属性編集）

## 主に扱う Item / Asset
* **Board 内では主に `model` 系 Item**を扱います。
* 画像アセット（サムネイルやテクスチャ）も一部関連して扱いますが、主体は単一の `.obj` / `.glb` などの 3Dモデル Asset です。

## 読むデータ
* `boards/{boardId}` コンテキスト情報。
* サブコレクション `boards/{boardId}/items/{itemId}` 内の `type == "model"` オブジェクト。
* 共有用の `publicModelIndex` (ギャラリー用データ)。

## 書くデータ
* `boards/{boardId}/items/{itemId}` に対して、単一の3Dモデル情報を追加・作成・更新・削除します。
* モデルに伴うサムネイル画像やアセット実体をStorageへ保存し、そのメタデータをItemへ書き込みます。

## Project / Board との関係
* 3DSSは、独自のデータベースや独自の Board 構造を持ちません。
* 他の子アプリ（3DSL等）と全く同じ **SEKKEIYA 共通の Board / Item 構造の上で動作** します。
* その Board 内に配置されている `model` Item を視覚的に一覧・管理するためのUIとして振る舞います。

## AI Drive との関係
* **モデルインポーター・出力連携**: 外部モデル（Rhino連携など）をインポートし、Asset 化して Board 内へ反映させるゲートウェイの役割を持ちます。
* 3DSSで変換・管理されたモデルは、AI Drive を通じて強力な検索対象となり、他アプリへシームレスに供給されます。

## AI Chat との関係
* AI に対して「特定のモデル」のタグ付けやカテゴリ自動整理、類似モデルの検索を指示できます。
* Chat に投げた画像を元に類似モデルを探すなどのモデルセントリックな会話の中心となります。

## 公開 / 非公開の扱い
* マイボード内では非公開（チーム内のみの権限次第）ですが、「全体公開設定」を行うことで `publicModelIndex` などのパブリックインデックスに登録され、一般公開されたアセット一覧に表示されるようになります。

## 将来の責務境界
* 3DSSの責務は、これ以上「複数モデルによる空間レイアウト」や「プレゼンテーション機能」へと拡張しません（それらは 3DSL や 3DSP の役割です）。
* 将来的にも3DSSは **1つのモデルに対する詳細管理、品質担保、インポート・エクスポートパイプラインのフロントエンド** として進化します。
